## 1. 背景与动机

Tailscale DERP 文档强调 DERP 只盲转已加密 WireGuard 包，无法解密。ZeroTier 也强调节点私钥不出设备。当前 Mesh 中，WebSocket/TLS 只保护传输层，server 内部仍能看到原始 IP 包头和内容。

## 2. 当前项目状态

- `internal/tunnel/server.go` 中 server 解析 IP 头 dst 做路由决策。
- 设备 secret 用于认证，不是包加密。
- 没有 session key / 端到端加密。

## 3. 实施方案

### 3.1 密钥模型

```text
device:
  long-term key (ed25519 / curve25519) — 注册时生成，私钥不出设备
  public key 上报控制面

session:
  短期 key (per peer pair) — 通过控制面协商或 DERP 交换
  周期性轮转
```

### 3.2 数据面包格式

```text
| version | dst_mesh_ip | nonce | encrypted_ip_packet | auth_tag |
```

- version：协议版本
- dst_mesh_ip：明文，relay 据此路由
- nonce：每次加密随机
- encrypted_ip_packet：AEAD 加密后的原始 IP 包
- auth_tag：完整性校验

### 3.3 协议握手

可选 Noise / WireGuard 风格握手，或简化版：

```text
1 client A 发送 hello + ephemeral public key
2 server 转发到 B
3 B 返回 hello + ephemeral public key
4 双方 derive shared secret = session_key
5 后续 packet 用 session_key 加密
```

### 3.4 与现有兼容

- 首字节版本号判别旧协议。
- 旧 client 仍可解密为明文。
- 新协议可渐进部署。

### 3.5 server 路由

server 仍可看 `dst_mesh_ip`，但看不到 IP payload：
- 路由决策可用。
- 抓包 / DPI 不可见业务内容。
- server 仍可统计 metadata（包数、字节数）。

## 4. 测试计划

- 单元：加密/解密、nonce、replay 保护。
- 集成：server 抓包看不到明文。
- 兼容：旧 client 互通。
- 性能：加密开销对延迟影响。
