## 1. 背景与动机

Tailscale 的核心价值不是“能连”，而是“谁能连谁”。Cloudflare One 更是以 Zero Trust policy 为中心。当前 Mesh 任何加入的设备都默认互通，缺乏身份驱动的细粒度策略。

## 2. 当前项目状态

- 设备只有 IP/secret，没有 tag 或 group。
- 转发路径不做策略匹配。
- 默认 allow。

## 3. 实施方案

### 3.1 身份模型

```text
device:
  id: ...
  name: ...
  tags: [admin, mac, dev]   # 多 tag
```

控制面提供：

```text
groups:
  admin: [device_id1, device_id2]
  server: [device_id3]
```

### 3.2 策略语法

```yaml
rules:
  - action: allow
    src: admin
    dst: server
    proto: tcp
    ports: [22, 443]
  - action: deny
    src: "*"
    dst: "*"
```

### 3.3 强制点

- 数据面：在转发前匹配策略；不匹配则丢。
- 控制面：注册时返回该设备的可见 peer 子集。

### 3.4 审计

- 控制面记录策略命中、未命中。
- 提供 `meshd acl` / API 查看。

## 4. 测试计划

- 单元：策略匹配逻辑。
- 集成：默认 deny 下任何未授权流量被拒。
- 边界：tag 集合、group 引用。
