## 1. 背景与动机

Tailscale 明确把 coordination server 作为控制面、DERP 与 WireGuard 隧道作为数据面。ZeroTier 把 VL1（传输）和 VL2（虚拟二层）拆开。Cloudflare One 客户端把 WARP tunnel、DoH、device orchestration 分成三个独立连接。

当前 `meshd` 同时承担注册、IP 分配、在线状态、TUN、WebSocket 中继、路由决策、API 等职责，模块边界不清，每加一个新能力（peer relay、ACL、MagicDNS）都会让代码继续膨胀。

## 2. 当前项目状态

- `cmd/meshd/main.go` 启动一个进程处理所有职责。
- `internal/tunnel/server.go` 同时负责 WebSocket、TUN、路由决策、设备业务。
- `internal/api/` 暴露 HTTP 接口。
- 没有清晰的“控制面 / 数据面”分层。

## 3. 实施方案

### 3.1 模块拆分

```text
internal/control/
  - 设备注册
  - IP 分配
  - peer 列表
  - policy 下发
  - gateway map 下发
  - key 协调

internal/relay/
  - WebSocket listener
  - 转发 IP/帧
  - 不参与设备业务

internal/path/
  - PathManager
  - 路径健康
  - 选择与切换

internal/api/
  - 控制面 HTTP API
```

### 3.2 接口

控制面调用数据面通过本地接口：

```go
type Relay interface {
    RegisterSession(ctx context.Context, info SessionInfo) (Session, error)
    Unregister(sessionID string) error
}
```

数据面通过事件通知控制面（设备上下线、统计）：

```go
type Events interface {
    OnSessionActive(info SessionInfo)
    OnSessionClosed(sessionID string)
    OnStats(info SessionStats)
}
```

### 3.3 部署形态

初期保留单进程多模块；接口清晰后可独立进程部署。
