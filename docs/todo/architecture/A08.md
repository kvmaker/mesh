## 1. 背景与动机

Tailscale、Cloudflare WARP 都支持把某个节点作为全局出口（`0.0.0.0/0`），让其他 peer 把所有流量转发到这个节点。常见场景：在外访问公司网络、统一出口审计、绕过地域限制。

## 2. 当前项目状态

- 设备没有 exit node 角色。
- 客户端默认路由指向物理网卡，不走 mesh TUN。

## 3. 实施方案

### 3.1 节点声明

```yaml
device:
  name: office-gateway
  exit_node: true
```

### 3.2 客户端启用

```bash
mesh up --exit-node office-gateway
```

### 3.3 路由切换

把系统默认路由切换到 mesh TUN：

```text
ip route add default dev mesh-tun
ip route add 10.100.0.0/24 dev mesh-tun  # mesh 网段
```

所有非 mesh 流量通过 mesh TUN，PathManager 转发到 exit node。

### 3.4 ACL 与审计

- exit node 必须配合 ACL，避免成为开放代理。
- 出口节点可记录流量审计。

## 4. 测试计划

- 单元：默认路由切换不破坏 mesh 网段路由。
- 集成：客户端通过 exit node 访问公网。
- 失败：exit node 不可达时回退到正常出口。
