## 1. 背景与动机

Tailscale、ZeroTier 都支持节点宣告子网路由，其他 peer 通过该节点访问背后 LAN。当前 Mesh 只能访问 mesh 网段，无法访问 peer 背后的真实子网。

## 2. 当前项目状态

- 设备表没有 advertised routes 字段。
- 客户端 TUN 路由表只包含 mesh 网段。
- 路由选择只识别 `IP → ClientConn`。

## 3. 实施方案

### 3.1 节点宣告

```yaml
device:
  name: home-nas
  advertised_routes:
    - 192.168.1.0/24
```

控制面审核后下发给其他 peer。

### 3.2 客户端路由

收到 advertised routes 后：

```text
ip route add 192.168.1.0/24 dev mesh-tun
```

流量到达 mesh TUN 后查 `PathManager`，目的为 advertised subnet 的包送往宣告节点。

### 3.3 ACL

- 默认 deny：未授权 peer 不能访问该子网。
- 控制面下发 ACL 规则允许/拒绝。

## 4. 测试计划

- 单元：subnet 路由发布、撤销。
- 集成：客户端 ping subnet 内设备。
- 安全：未授权 peer 被拒绝。
