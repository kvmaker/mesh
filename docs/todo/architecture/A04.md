# A04. LAN direct：同局域网按需直连 — 详细分析

## 实施记录

（实施后填写：分支、改动文件、关键设计、测试结果）

## 1. 背景与动机

ZeroTier、Tailscale 都优先尝试局域网直连，避免跨广域网。当前 Mesh 即使两个 client 处于同一 LAN，也强制走 server 中继，浪费延迟和带宽。

## 2. 当前项目状态

- 设备注册不包含 LAN endpoint 候选。
- 转发路径只识别 server relay。
- 没有按需探测机制。

## 3. 实施方案

### 3.1 上报 LAN endpoint

设备在控制面注册时附带 LAN 候选：

```text
{
  "lan_endpoints": [
    "192.168.1.8:51820",
    "10.0.0.5:51820"
  ]
}
```

控制面转发给其他 client。

### 3.2 PathManager 新增 LAN direct

```go
type Path struct {
    Type PathType  // relay | peer_relay | lan_direct | direct
    Endpoint string
    ...
}
```

### 3.3 按需探测

只有当某 peer 实际有持续流量（最近 N 秒超过 M 包），才发起 LAN direct probe：

```text
probe:
  udp packet to peer.lan_endpoint
  expect ack within timeout
on success:
  promote to primary path
on failure:
  fall back to relay
```

### 3.4 隐私

- LAN endpoint 默认只在双方都开启 LAN direct 时才公开。
- 可配置只对白名单 IP 段开放。

## 4. 测试计划

- 单元：探测成功率与失败处理。
- 集成：两台 client 在同一 LAN，ping 跨 server 与 LAN direct 的延迟对比。
- 失败模拟：LAN 不可达时回退到 server。

- 默认 `lan_direct.enabled: false`。
- 用户显式开启。
- 默认按需探测，避免一上线就全网广播。
