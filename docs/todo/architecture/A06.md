## 1. 背景与动机

Tailscale MagicDNS 让 mesh 内设备可以直接用 `device-name.mesh` 互相访问。当前 Mesh 只能通过 IP 访问，体验较差。

## 2. 当前项目状态

- 设备名仅在 API 列表中。
- 客户端没有 DNS 拦截或代理。
- TUN 仅承载 IP 流量。

## 3. 实施方案

### 3.1 域名生成

控制面根据 device name + network cidr 派生全限定域名：

```text
macbook → macbook.mesh
nas      → nas.mesh
```

### 3.2 DNS 通道

- 客户端拦截 system DNS 解析（`*.mesh` 走 mesh 内部 DNS）。
- 内部 DNS 通道：
  - DoH over control plane（最简单，HTTPS 即可）；
  - 或 UDP/53 over mesh tunnel（更原生，但需要端口管理）。

### 3.3 缓存

- 客户端本地缓存域名 → IP。
- TTL 由控制面下发。
- 设备 online 状态变化时主动失效。

### 3.4 隐私与安全

- DNS 响应包含 mesh IP，不暴露公网 IP。
- 查询应当认证，避免被截获重放。

## 4. 测试计划

- 单元：域名派生、缓存。
- 集成：客户端 `ping macbook.mesh` 成功。
- 状态联动：设备下线时缓存立即失效。
