## 1. 背景与动机

当前 client 只配置单一 `ServerDomain`，单 gateway 中继。Tailscale DERP map、ZeroTier roots+moons 都使用多中继节点地图，client 基于延迟选择 home relay，relay 失败时 failover。Cloudflare 也通过 Anycast 让最近的 PoP 处理流量。

## 2. 当前项目状态

- `internal/client/config.go` 仅保存 `ServerDomain` 等字段。
- 没有 gateway map、没有健康探测、没有就近选择。

## 3. 实施方案

### 3.1 配置演进

```json
{
  "gateways": [
    {"id": "ap-shanghai",  "url": "wss://sh.example.com/tunnel", "region": "cn-east",  "priority": 10},
    {"id": "ap-guangzhou","url": "wss://gz.example.com/tunnel", "region": "cn-south", "priority": 20}
  ]
}
```

保留旧 `server_domain` 兼容：自动转换为单元素 `gateways`。

### 3.2 控制面下发

注册响应携带 `gateway_map`，包含所有可用 gateway：

```text
{
  "device_id": "...",
  "device_ip": "10.100.0.2",
  "peers": [...],
  "gateway_map": [...]
}
```

### 3.3 健康探测

- 周期性测量每个 gateway 的 RTT、丢包率、队列深度。
- 选 home relay + 1-2 个 backup。
- 失败时自动切换。

### 3.4 集成到 PathManager

`PathManager.RegisterRoute` 接收每 gateway 作为 `relay` path 类型，由选择策略决定使用哪个。

## 4. 测试计划

- 单测：home/backup 选择。
- 故障模拟：home gateway 不可达，切换到 backup。
- 配置兼容：旧 `server_domain` 仍能运行。
