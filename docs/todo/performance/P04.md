## 1. 背景与动机

没有指标就无法判断转发优化的真实效果，也无法定位线上问题。当前 `ClientConn` 和 `TunnelClient` 只有基础 `Tx/Rx/Bytes`，缺少 `queue_depth`、`drop_packets`、`rtt`、`last_active`、server 全局错误计数等关键指标。

## 2. 当前项目状态

- `internal/tunnel/router.go` 中 `ConnStats` 仅含 `Tx/Rx/Bytes/LastPacket`。
- `internal/tunnel/stats.go` 客户端侧 `ClientStats` 同样缺少。
- `internal/api/handler_stats.go` 直接转发 router stats。
- `internal/client/status.go` 客户端状态显示字段有限。

## 3. 实施方案

### 3.1 per peer 指标

```go
type ConnStats struct {
    DeviceID      string
    IP            string
    ConnectedAt   time.Time
    TxPackets     uint64
    TxBytes       uint64
    RxPackets     uint64
    RxBytes       uint64
    DropPackets   uint64
    QueueDepth    int64
    QueueMaxDepth int64
    LastRTTMs     int64
    LastActive    time.Time
}
```

### 3.2 server 全局指标

```go
type ServerStats struct {
    ActiveClients     int
    RouteMiss         uint64
    WriteErrors       uint64
    TUNReadErrors     uint64
    WSReadErrors      uint64
    DropsTotal        uint64
    FramesByType      map[string]uint64
}
```

### 3.3 展示

- `mesh status`：客户端视角，显示 home relay、queue depth、drops、最近 RTT、最近活跃。
- `meshd stats`：服务端视角，显示每个 peer 的 ConnStats、ServerStats 摘要。
- API：`/api/stats` 暴露结构化 JSON。

## 4. 测试计划

- 单测：stats 字段填充正确。
- API 测试：JSON 字段稳定。
- 集成：转发异常时 drops 与 errors 同步增长。
